# Discovery Stream API Contract

> Generated by PM Agent | Date: 2026-04-07

## Base URL
`/api/discovery`

## Authentication
All endpoints require JWT Bearer token.

---

## Endpoints

### GET /api/discovery/search/stream
- **Description**: SSE stream of search pipeline progress. Real-time stage events as the multi-agent search pipeline executes.
- **Auth**: Required (Bearer token via `Authorization` header — frontend uses `fetch()`, not `EventSource`)
- **Rate Limit**: 20 requests/minute per user
- **Content-Type**: `text/event-stream`
- **Query Params**:
  | Param | Type | Required | Description |
  |-------|------|----------|-------------|
  | preset_id | UUID | yes | Classroom preset ID |
  | query | string (1-500 chars) | yes | Search query |

- **SSE Event Format**:
  ```
  event: stage
  data: <SearchStageEvent JSON>
  ```

- **Event Sequence (normal flow)**:
  ```
  { "stage": "query_generation",  "status": "working" }
  { "stage": "query_generation",  "status": "done", "data": { "ddgs": [...], "youtube": [...], "openalex": [...] } }
  { "stage": "federated_search",  "status": "working" }
  { "stage": "federated_search",  "status": "done", "data": { "counts": { "ddgs": 5, "youtube": 4, "openalex": 3 } } }
  { "stage": "rag_preparation",   "status": "working" }
  { "stage": "rag_preparation",   "status": "done" }
  { "stage": "evaluation",        "status": "working", "resource_url": "https://..." }
  { "stage": "adversarial",       "status": "working", "resource_url": "https://..." }
  { "stage": "evaluation",        "status": "done",    "resource_url": "https://..." }
  { "stage": "adversarial",       "status": "done",    "resource_url": "https://..." }
  { "stage": "complete",          "status": "done", "data": <EvaluatedSearchResponse> }
  ```

- **Event Sequence (cache hit)**:
  ```
  { "stage": "complete", "status": "done", "cached": true, "data": <EvaluatedSearchResponse> }
  ```

- **Errors**:
  | Code | Condition |
  |------|-----------|
  | 401 | Missing or invalid Bearer token |
  | 404 | Preset not found or not owned by user |
  | 422 | Invalid query params |
  | 429 | Rate limit exceeded |
  | 502 | All search providers failed |

## Data Models

### SearchStageEvent
| Field | Type | Required | Description |
|-------|------|----------|-------------|
| stage | enum | yes | `query_generation` \| `federated_search` \| `rag_preparation` \| `evaluation` \| `adversarial` \| `complete` |
| status | enum | yes | `working` \| `done` |
| resource_url | string | no | Resource URL (only for evaluation/adversarial per-resource events) |
| cached | bool | no | `true` when result is from cache (only on `complete`) |
| data | object | no | Stage-specific payload (only on `done` events) |

### Existing endpoint (unchanged)

`GET /api/discovery/search` — Returns `EvaluatedSearchResponse` synchronously. Retained for backward compatibility and SSE fallback.

---

## Notes
- SSE connection uses `fetch()` + `ReadableStream` on the frontend (not `EventSource`) to support `Authorization` header
- Stream terminates after `complete` event
- Frontend should implement `AbortController` for cleanup on unmount/re-search
- Existing `GET /search` endpoint is NOT deprecated — serves as fallback
