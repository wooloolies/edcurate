## Design Hierachy

Designers apply eight key visual hierarchy principles to help the mind grasp ideas quickly.

1. Alignment refers to the placement of one visual element in relation to the others in a group. When objects are aligned, or lined up, they're understood as related to one another.

2. Color is often described in terms of hue, lightness, and saturation. Color can be tricky, though, because people perceive it so differently. To establish a clear visual hierarchy, Cardona specifies luminance and brilliance in design. It is recommended to use a pre-set color palette with good contrast, which can be easily set using a color wheel generator.

3. Contrast is when notably different elements are close together. Designers often juxtapose warm and cool elements or complementary colors to enhance visual appeal and boost contrast ratios for accessibility.

4. Proximity describes how near or far items are in relation to each other. Grouping or “chunking” objects is an example of this principle in action.

5. Size refers to the dimensions of an object or design element. The size of UI elements like buttons or content can be an important consideration for accessibility. “If you don't provide low-vision users the opportunity to have agency over text and image size, you're making your content difficult to read and understand,” Cardona says.

6. Texture is the way a surface feels or a UI element is perceived to feel. "Texture allows you to ascribe meaning to design, and may help give a clear visual indication of how something is to be used," says Cardona. When UI designers use texture to mimic the appearance of tactile, real-life material, that's referred to as skeuomorphism—and it can come across as dated or literal unless handled carefully. "With digital design, a lot of designers avoid texture—but it can be really impactful,” Cardona explains. “Texture can be used to create highlights, instead of relying on color."

7. Time unfolds on screens differently than on printed pages and static products. Screens can change, react, and transform, opening up a new world of temporal design. 

## Project Context

Edcurate is:

> A transparent search and adaptation engine designed to help teachers discover Year 7–12 resources, evaluate curated collections, and synthesize them into ready-to-teach slides, drastically reducing the workload of researching and summarizing.

Core focus areas:

- Context-aware discovery
- Transparent evaluation
- Slide generation based on evaluation of a collection folder

Problem statement:

- Generic AI search results often do not match highly specific classroom needs.
- Teachers struggle to trust how resources are selected, evaluated, and adapted.
- Teachers do not mainly struggle to find resources; they struggle to trust, evaluate, and adapt them for their classroom.

Every design decision should reinforce these qualities:

- Trustworthy
- Transparent
- Intelligent
- Focused

The visual system should feel more like a professional teaching assistant than a flashy AI product.

## Design Principles

### 1. Minimal but not static

- Keep layouts clean, spacious, and easy to scan.
- Use motion, layering, density shifts, hover states, and progressive disclosure to create energy.
- Make the product feel alive without adding noise.

### 2. Clarity over decoration

- Prioritize readable hierarchy, obvious states, and transparent information flow.
- Use motion and styling to explain system behavior, not to distract.
- Avoid generic AI visuals, visual clutter, and unnecessary effects.

### 3. No default gradient backgrounds

- Prefer solid backgrounds, subtle surface separation, tonal layering, borders, and shadows.
- Only use gradients when they are explicitly requested and serve a clear purpose.

### 4. Strong typography system

- Define one main font for body copy, one sub-font if needed for supporting UI text, and one title font for headings or emphasis.
- Keep typography usage consistent across pages and components.
- Use typography to build confidence, editorial clarity, and a research-grade tone.

### 5. Palette with purpose

- Identify and explain the palette before or during major design work when useful.
- Map colors to semantic roles such as background, surface, primary action, emphasis, success, warning, and danger.
- Favor calm, credible palettes that support educational and analytical workflows.

### 6. Motion with meaning

- Use motion to show transitions, feedback, focus shifts, loading states, ranking changes, and reveal logic.
- Keep animation intentional, smooth, and lightweight.
- Respect accessibility and reduced-motion expectations.

## Working Style

### When the prompt is unclear

- Actively ask clarifying questions.
- Clarify visual tone, audience, references, target platform, and desired level of motion.
- Do not guess when a missing detail could materially change the design direction.

### When the user provides reference images

- Commit closely to the references when appropriate.
- Identify what should be preserved from the references: composition, spacing, typography mood, palette, interaction style, or motion language.
- Adapt the reference style to Edcurate's product goals instead of copying blindly.

### When exploring design directions

- Research relevant styles based on the user's description.
- Suggest a small number of distinct directions with rationale.
- Explain why each direction fits Edcurate's goals of transparency, trust, and reduced teacher workload.

## Recommended Output for Design Tasks

When giving design recommendations, try to cover:

- Visual direction
- Typography pairing
- Color palette and semantic roles
- Motion behavior
- Component mood
- Layout density and spacing
- Why the direction fits Edcurate

## Edcurate-Specific Heuristics

- Discovery screens should feel guided, contextual, and research-oriented.
- Evaluation screens should feel transparent, inspectable, and evidence-based.
- Slide-generation screens should feel efficient, polished, and outcome-focused.
- Teacher-facing interfaces should reduce cognitive load and support quick judgment.
- The UI should communicate that the system helps educators make better decisions, not opaque decisions.
